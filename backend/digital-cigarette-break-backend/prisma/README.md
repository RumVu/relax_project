# Prisma Workspace

This project keeps Prisma schema source files split by domain in `prisma/schema-parts/`.

Prisma CLI configuration lives in `prisma.config.ts`.

## Source of truth

- Edit `prisma/schema-parts/*.prisma.part`
- Do not edit `prisma/schema.prisma` by hand
- `prisma/schema.prisma` is generated by `prisma/scripts/build-schema.cjs`

## Common commands

- `npm run prisma:sync-schema`
- `npm run prisma:validate`
- `npm run prisma:generate`
- `npm run prisma:db:push`
- `npm run prisma:migrate:dev -- --name init`
- `npm run prisma:migrate:deploy`
- `npm run prisma:seed`
- `npm run prisma:studio`

`prisma:validate` and `prisma:generate` use a safe local fallback `DATABASE_URL` if one is not set yet.
Database commands that actually connect to a DB still expect a real `DATABASE_URL`.

## Suggested workflow

1. Edit one or more files in `prisma/schema-parts/`
2. Run `npm run prisma:validate`
3. Run `npm run prisma:generate`
4. Create a migration with `npm run prisma:migrate:dev -- --name <migration_name>`

## Seed data

The seed script is at `prisma/seed.cjs`.

It currently creates:

- subscription tiers
- tier features
- tier limits
- achievements
- badges
- sample quotes
